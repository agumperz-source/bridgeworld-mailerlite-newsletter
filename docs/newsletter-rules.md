# Newsletter Rules

Editorial and publishing rules for Bridgeworld newsletter output.

## Template Families

The current production template is Template Family 1: Classic Editorial.

Future redesigns and audience-specific newsletters should be added as separate template families. They should reuse shared validated components for bridge diagrams, auctions, action-score tables, MailerLite tokens, Stay Connected, and footer behavior.

## Reference Library

Historical tested newsletters are recorded in `docs/newsletter-reference-findings.md`.

Those references guide design choices but do not override the current canonical shell or validation gates.

## Component Rules

Shared auction, footnote, bidder-name, scoring-table, hand-diagram, Stay Connected, and footer rules are recorded in `docs/component-style-rules.md`.
